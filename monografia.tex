%resumo
\begin{resumo}
O trânsito nas metrópoles apresenta problemas desafiadores para o planejamento urbano e é um dos fatores que pode alterar significativamente a qualidade de vida dos seus habitantes. O transporte público e os meios de locomoção não motorizados são amplamente citados como alternativas de mobilidade ambiental e socialmente mais corretas, mas frequentemente o usuário não sabe qual a melhor forma de utilizá-los devido às complexas redes de transporte existentes nas grandes cidades.
Com o intuito de facilitar e incentivar o uso do transporte coletivo, este trabalho apresenta o desenvolvimento de uma ferramenta web para realizar planejamento de rotas atraves de transporte público. São discutidos aspectos teóricos sobre a abordagem de redes multimodais e o algoritmo de menor caminho utilizado, bem como o processo de engenharia de \emph{software} e as ferramentas empregadas no desenvolvimento, do banco de dados de grafos ao sistema de controle de versão. Apresenta-se tambem a metodologia de trabalho da equipe e uma descrição do produto final.
Resultou deste projeto um \emph{software} livre, aberto a contribuições externas, capaz de resolver consultas de menor caminho através de um serviço \emph{web}, o qual pode ser utilizado por outras aplicações, assim como uma interface \emph{web} para o usuário final.
\end{resumo}

%abstract
\begin{abstract}
Traffic in the big cities brings challenging problems for urban planning and is one of the factors that can significantly alter the quality of life of its citizens. Public transport and nonmotorized transportation are widely cited as alternatives to mobility that are environmentally and socially more friendly, but the user frequently doesn't know the best way to use them because the big cities have complex transportation networks.
With the intention to help incentivate and facilitate the use of public transport, this work presents the development of a web tool that plans routes using public transport. It includes a discussion of some theoretical aspects regarding the multi-modal approach and the shortest path algorithm, and also of the software engineering process and the development tools used, the graph database and the version control system. The work methodology used by the team and a description of the final product are also presented.
The result of this project was a free software, open to external collaborations, capable of solving shortest path queries through a web service, which can be used by other applications, and with a web interface for the final user.
\end{abstract}
